Problem Statement #22 — Interview Trainer Agent
Problem Statement
The Challenge

Every year, thousands of students and freshers fail job interviews — not due to lack of knowledge, but due to lack of structured preparation.

What's Missing Today
  • Generic YouTube videos & static PDFs don't adapt to individual skill sets
  • No role-specific or company-specific question generation
  • No real-time answer evaluation with scoring
  • No confidence tracking or personalised study roadmap
The Objective

Build an AI-powered conversational Interview Trainer Agent that:

  • Conducts structured mock interviews tailored to role, company & experience
  • Evaluates answers in real-time with score + feedback
  • Tracks confidence growth from start to end
  • Generates a 7-day personalised study roadmap
  • Runs across all major IT companies: TCS, Infosys, Wipro, Accenture, Cognizant

Key Design: Server-Side Sessions

Frontend sends only session_id + message — no history in HTTP body. Backend holds all history, trims to a rolling window, sends fixed-size context to LLM. Token cost stays flat regardless of interview length.

Request Flow
1User clicks Start → Streamlit calls /session/new → gets UUID
2Each message → POST /chat with {session_id, message}
3FastAPI retrieves session history, builds trimmed LLM message list
4LLM responds → sanitized → appended to session → returned to UI

Key Learnings & Future Scope
Key Learnings & Future Scope
🧠 Key Learnings
  • Prompt engineering is critical — ARIA's behaviour quality depends entirely on how precisely the system prompt defines each rule
  • Structural role boundaries beat text labels — using system/user/assistant roles instead of fake "ARIA:"/"Candidate:" labels eliminated hallucination
  • Token budget management must be architectural, not just prompt-level — server-side sessions were the key architectural fix
  • Dynamic output caps per phase are far more efficient than a fixed cap for all turns
  • IBM Cloud Lite provides a complete free AI stack — watsonx.ai + IBM SDK together at zero cost
🚀 Future Scope
  • Resume Upload — parse CV and generate personalised questions from actual projects and skills listed
  • Multilingual Support — conduct interviews in Hindi, Tamil, Telugu for regional candidates
  • Redis Session Store — replace in-memory dict for multi-worker production deployment
  • ARIA TTS (Text-to-Speech) — ARIA speaks the questions aloud for a true interview simulation
  • Company-specific RAG — upload TCS/Infosys PDFs to knowledge base for hyper-accurate questions
  • Analytics Dashboard — track score trends across multiple mock interview sessions over time
